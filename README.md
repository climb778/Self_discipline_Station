# 自律小站

一款面向个人的自律打卡、任务管理、专注记录和云笔记应用。帮助你规划每日待办、专注执行、复盘成长、记录学习笔记。

前端基于 uni-app + Vue 3，后端基于 Spring Boot + MySQL。支持 H5 浏览器访问和 Android App 两种使用方式。

## 技术栈

**前端：**
- uni-app + Vue 3 Composition API (`<script setup>`)
- Vite
- HBuilderX / Android App 云打包

**后端：**
- Spring Boot
- MyBatis-Plus
- MySQL
- JWT 登录认证
- 文件上传（笔记附件）

## 核心功能

| 模块 | 功能 |
|------|------|
| 每日待办 | 每日任务列表、完成率统计、待办提醒卡片 |
| 岁月打卡 | 日历视图、按月查看完成状态、每日任务详情 |
| 过往记录 | 搜索筛选、已完成/长期/过期/归档分类 |
| 我的主页 | 用户资料、行动仪表盘、7天趋势图、快捷入口 |
| 专注模式 | 番茄钟 15/25/45 分钟或自定义时长、暂停/继续/结束 |
| 学习记录 | 学习日志、科目筛选、掌握程度、关联计划/任务 |
| 数据报告 | 周报/月报、趋势图、分类统计、报告海报 |
| 目标计划 | 计划 CRUD、模板、任务关联、进度统计 |
| 智能计划 | 目标输入、本地规则生成、阶段安排 |
| 设置中心 | 个人信息、主题换肤、服务器状态检测、数据管理 |
| **云笔记** | Markdown 编辑器、双向链接、标签/文件夹、附件上传 |
| **云笔记编辑** | 草稿自动保存、未保存返回提醒、Markdown 工具栏、保存状态提示 |
| **备份/导入** | 笔记 JSON 备份导出/恢复、Markdown 导入、H5 和 App 双端支持 |

## 运行方式

### 前端开发（H5）

```bash
npm install
npx uni -p h5
```

浏览器访问 http://localhost:5173

### 前端构建（H5）

```bash
npx uni build -p h5
```

### 后端运行

```bash
cd backend
mvn spring-boot:run
```

### 后端环境变量

生产部署时通过环境变量配置，不要将真实密码写入仓库：

| 变量 | 说明 | 示例 |
|------|------|------|
| `DB_HOST` | 数据库地址 | `127.0.0.1` |
| `DB_PORT` | 数据库端口 | `3306` |
| `DB_NAME` | 数据库名 | `self_discipline_station` |
| `DB_USER` | 数据库用户名 | `selfapp` |
| `DB_PASSWORD` | 数据库密码 | （生产环境设置） |
| `JWT_SECRET` | JWT 签名密钥 | （生产环境设置） |
| `CORS_ALLOWED_ORIGINS` | CORS 允许的来源 | `https://your-domain.com` |
| `UPLOAD_PATH` | 文件上传目录 | `/data/uploads/notes/` |

## Android 打包

1. 使用 HBuilderX 打开项目根目录
2. 菜单栏选择 **发行 -> 原生 App-云打包**
3. 选择 Android 平台，使用项目已配置的 AppID 和证书
4. 等待打包完成，将 APK 传到手机安装

注意：
- APK 文件不提交到源码仓库，如需发布可使用 GitHub Release
- 最低支持 Android 5.0（API 21）
- 应用已配置为竖屏模式

## 部署说明

- 后端可部署到 Linux 服务器
- 通过 Nginx 反向代理 `/api/` 到后端端口
- 可配合 Cloudflare Tunnel 暴露 HTTPS API
- 生产环境应使用环境变量配置数据库和 JWT 密钥
- 不要把生产密码写进代码仓库

## 数据说明

- 本地自律数据（待办、打卡、专注记录等）存储在本地
- 云笔记数据存储在后端数据库，需要登录账号使用
- 附件存储在服务器上传目录
- 支持笔记备份导出为 JSON 文件，可随时恢复
- 建议定期通过笔记页的「备份」功能导出备份

- 保存状态实时提示（修改中 / 草稿已保存 / 保存中 / 已保存 / 失败）

**App 基座备份导入适配：**
